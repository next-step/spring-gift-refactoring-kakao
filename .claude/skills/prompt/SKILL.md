# Prompt Documenter — 프롬프트 누적 문서화 스킬

## 핵심 원칙

> **`docs/PROMPT.md` 파일 하나에 시간순으로 누적 기록한다.**
> 매 프롬프트마다 새 문서를 생성하지 않는다.

## 워크플로우

### 1단계: docs/PROMPT.md 파일 확인

프롬프트 문서화 요청을 받으면 **가장 먼저** 기존 `docs/PROMPT.md` 파일이 있는지 확인한다.

- Read 도구로 `docs/PROMPT.md`를 읽는다.
- **파일이 있는 경우**: 기존 내용을 읽어온 뒤, Edit 도구로 새 프롬프트 항목을 하단에 추가(append)한다.
- **파일이 없는 경우**: Write 도구로 헤더 섹션부터 새로 생성한다.

### 2단계: docs/PROMPT.md에 기록

#### 파일 전체 구조 (최초 생성 시)

```markdown
# Prompt Documentation Log

> 프롬프트 누적 기록 문서
> 최초 생성: YYYY-MM-DD
> 최종 수정: YYYY-MM-DD
> 총 기록 수: N건

---

## 목차

| # | 제목 | 기록 일시 |
|---|------|-----------|
| 1 | 코드 리뷰어 프롬프트 | 2025-06-15 14:30 |

---

## [#1] 코드 리뷰어 프롬프트
> YYYY-MM-DD HH:MM (KST)

\```
{프롬프트 전문}
\```

---
```

#### 각 프롬프트 항목 포맷

새 프롬프트를 추가할 때 아래 포맷을 따른다:

```markdown
---

## [#N] {프롬프트 제목}
> YYYY-MM-DD HH:MM (KST)

\```
{프롬프트 전문}
\```
```

### 3단계: 헤더 및 목차 업데이트

새 항목을 추가한 후 반드시 아래를 업데이트한다:

1. **최종 수정 날짜** — 현재 날짜로 변경
2. **총 기록 수** — 카운트 +1
3. **목차 테이블** — 새 행 추가

## 구현 절차 (Claude가 따를 순서)

1. Read 도구로 `docs/PROMPT.md` 확인
2. 파일이 있으면 기존 내용을 읽고, 없으면 Write 도구로 새 헤더 생성
3. 다음 번호(#N) 결정 — 기존 파일의 마지막 번호 + 1
4. Edit 도구로 항목 포맷에 맞춰 파일 하단에 append
5. Edit 도구로 헤더(최종 수정일, 총 기록 수)와 목차 테이블 업데이트

## 여러 프롬프트 한꺼번에 처리

사용자가 여러 프롬프트를 동시에 제공하면:
1. 각각 별도 항목(#N, #N+1, ...)으로 분리 기록
2. 모두 같은 타임스탬프 사용 (동시 입력이므로)

## 주의 사항

- 프롬프트에 민감 정보(API 키, 비밀번호 등)가 포함되면 경고 표시 후 마스킹 여부를 사용자에게 확인
- 기존 항목은 절대 수정하지 않는다 — append only
- 타임스탬프는 항상 KST 기준
