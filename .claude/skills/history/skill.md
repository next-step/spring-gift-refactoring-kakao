프로젝트의 중요한 변경사항을 `./history/` 디렉토리에 마크다운 파일로 기록하고, `History.md`를 인덱스로 관리한다.

## 활성화 조건

사용자가 다음과 같은 요청을 할 때 이 스킬을 실행:
- "히스토리 기록해줘" / "변경사항 기록해줘"
- "history 남겨줘" / "히스토리 업데이트"
- "작업 내용 정리해줘" / "오늘 작업 기록"
- 큰 작업 단위가 완료된 후 "기록 남겨줘"

## 수행 방법

### 핵심 원칙

- **큰 흐름 단위로만** 기록한다. 사소한 오타 수정, 변수명 변경 같은 미세한 변경은 기록하지 않는다.
- 하나의 파일에는 하나의 주제/작업 흐름을 담는다.
- 간결하고 핵심만 전달하는 문체를 사용한다.

### 기록 절차

1. **변경사항 파악**: `git diff`, `git log`, 또는 현재 대화 맥락에서 어떤 변경이 있었는지 파악한다.
2. **주제 결정**: 변경사항의 핵심 키워드를 한두 단어로 요약한다 (예: `인증-구현`, `상품-API-리팩토링`, `위시리스트-추가`).
3. **히스토리 파일 생성**: `./history/` 디렉토리에 아래 명명 규칙에 따라 파일을 생성한다.
4. **History.md 업데이트**: 프로젝트 루트의 `History.md`에 새 항목을 추가한다.

### 파일 명명 규칙

```
./history/yyyy-mm-dd-H-mm-키워드.md
```

- 현재 시각 기준 (24시간 형식)
- 키워드는 한글 또는 영어 소문자, 하이픈(`-`)으로 연결
- 예시: `2026-02-26-14-30-인증-구현.md`, `2026-02-26-9-15-product-api.md`

### 히스토리 파일 (개별) 형식

```markdown
# [주제 제목]

> 작성일: yyyy-mm-dd HH:mm

## 배경

[왜 이 작업을 했는지 간단히]

## 변경 내용

- [핵심 변경사항 1]
- [핵심 변경사항 2]
- ...

## 주요 파일

- `경로/파일명` - 간단한 설명
- ...

## 비고

[선택사항: 추후 참고할 점, 남은 과제 등]
```

- 모든 섹션을 채울 필요 없음. **배경**과 **변경 내용**은 필수, 나머지는 필요할 때만 작성.

### History.md 형식

프로젝트 루트에 `History.md` 파일을 관리한다. 이 파일은 전체 프로젝트 개선 히스토리의 개요이자, 개별 히스토리 파일로의 바로가기 모음이다.

```markdown
# 프로젝트 히스토리

프로젝트 변경 히스토리 인덱스.

## 타임라인

### yyyy-mm-dd

- [간단한 변경 요약](./history/yyyy-mm-dd-H-mm-키워드.md)

### yyyy-mm-dd

- [간단한 변경 요약](./history/yyyy-mm-dd-H-mm-키워드.md)
- [간단한 변경 요약](./history/yyyy-mm-dd-H-mm-키워드.md)
```

- 같은 날짜의 항목은 하나의 날짜 헤더 아래에 모은다.
- 최신 항목이 **위**에 오도록 역순 정렬한다.
- 링크 텍스트는 핵심 내용을 한 줄로 요약한 것이어야 한다.

### 디렉토리 구조

```
프로젝트 루트/
├── History.md              ← 인덱스 (바로가기 모음)
└── history/
    ├── 2026-02-26-14-30-인증-구현.md
    ├── 2026-02-26-16-00-상품-API.md
    └── ...
```

### 주의사항

- `./history/` 디렉토리가 없으면 먼저 생성한다.
- `History.md`가 없으면 새로 생성한다. 이미 있으면 기존 내용을 유지하며 새 항목만 추가한다.
- 현재 시각은 `date` 명령어로 확인한다.
- 같은 날짜 헤더가 이미 `History.md`에 있으면 해당 헤더 아래에 항목을 추가한다.
