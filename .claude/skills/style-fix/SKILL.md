# 1단계: 스타일 정리

## 목표
프로젝트 전반의 스타일 불일치를 찾아 일관되게 정리한다. **작동이 바뀌지 않아야 한다.**

## 대상 범위
`src/main/java/gift/` 아래의 모든 `.java` 파일

## 정리 항목 및 기준

### 1. `@Autowired` 제거
- Spring 4.3+ 이후 생성자가 하나면 `@Autowired` 불필요
- 모든 생성자 주입에서 `@Autowired` 어노테이션을 제거한다

### 2. `var` vs 명시적 타입 통일
- 로컬 변수는 **명시적 타입**으로 통일한다
- `var member = ...` → `Member member = ...`

### 3. `final` 키워드 통일
- 로컬 변수에 `final`을 **사용하지 않는** 방향으로 통일한다
- `final Member member = ...` → `Member member = ...`
- 필드의 `private final`은 그대로 유지한다

### 4. 에러 메시지 언어 통일
- 예외 메시지를 **한글**로 통일한다
- 예: `"Email is already registered."` → `"이미 등록된 이메일입니다."`

### 5. 필드 사이 빈 줄 통일
- Entity 필드 사이의 불필요한 빈 줄을 제거한다
- 어노테이션이 붙은 필드 그룹 앞에만 빈 줄을 둔다

### 6. 주석 스타일 통일
- `/* */` 블록 주석을 `/** */` JavaDoc 주석으로 통일한다 (클래스 레벨)
- 불필요한 주석(코드만으로 의미가 명확한 경우)은 제거한다

### 7. import 정렬
- 순서: `gift` → `jakarta` → `org` → `java` (알파벳순)
- 사용하지 않는 import 제거

## 작업 방식

1. 한 정리 항목씩 전체 파일에 적용한다
2. 각 항목 적용 후 `./gradlew build`로 빌드가 깨지지 않는지 확인한다
3. 작동 변경이 없는지 반드시 검증한다
4. 항목별로 커밋 메시지를 제안한다

## 제약
- 메서드 시그니처, 로직, 반환값을 변경하지 않는다
- 새로운 클래스나 메서드를 추가하지 않는다
- 테스트를 비활성화하거나 삭제하지 않는다
