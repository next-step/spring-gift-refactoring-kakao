# 코드 스타일 정리 및 구조 리팩토링 (Phase 1)

> 작성일: 2026-02-26 15:36

## 배경

기존 코드베이스의 일관성을 높이고 불필요한 보일러플레이트를 제거하기 위한 Phase 1 리팩토링 수행.

## 변경 내용

- 불필요한 `@Autowired` 어노테이션 제거 (생성자 주입 시 자동 적용)
- 불필요한 클래스 레벨 주석 제거 (`member`, `auth`, `option`)
- `collect(Collectors.toList())` → `toList()`로 모던 Java 스타일 적용
- 숫자 HTTP 상태 코드 → `HttpStatus` 상수로 교체 (`wish`, `order`)
- `@RequestMapping`의 명시적 `path` 속성 제거 (`auth`, `option`)
- 회원 도메인 에러 메시지 한국어 통일
- 중복 `@ExceptionHandler`를 `GlobalExceptionHandler`로 추출

## 주요 파일

- `src/main/java/gift/config/GlobalExceptionHandler.java` - 신규: 전역 예외 처리
- `src/main/java/gift/auth/KakaoAuthController.java` - 어노테이션 정리
- `src/main/java/gift/option/OptionController.java` - 스타일 정리, 중복 핸들러 제거
- `src/main/java/gift/member/AdminMemberController.java` - 어노테이션/에러 메시지 정리
- `src/main/java/gift/member/MemberController.java` - 어노테이션/에러 메시지 정리
- `src/main/java/gift/wish/WishController.java` - HttpStatus 상수 적용
- `src/main/java/gift/order/OrderController.java` - HttpStatus 상수 적용
- `src/main/java/gift/product/ProductController.java` - 중복 핸들러 제거
